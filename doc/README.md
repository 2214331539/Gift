## 推荐阅读顺序

1. 产品、设计、开发先阅读 `01`，统一“Gift 不是导购站，而是真实礼物故事社区”的方向。
2. 技术负责人阅读 `02`、`06`、`09`，确认架构和工程边界。
3. 前端阅读 `03`、`04`、`05`，按页面和 API 拆任务。
4. 后端阅读 `05`、`06`、`08`，按接口、数据和审核流程实现。
5. 测试和运营阅读 `07`、`08`、`10`，准备标签、内容、验收和上线检查。

## 全局开发假设

- 第一版目标是 Web MVP，不做移动端 App。
- 核心内容是用户真实送礼案例，不以商品购买链接为核心。
- MVP 需要登录注册、发布礼物、图片上传、搜索筛选、收藏、评论、匿名发布、基础后台审核。
- 暂不做私信、关注、复杂积分、视频、电商闭环、复杂 AI 推荐。
- 上线前需要准备 100-300 条高质量种子礼物案例，避免 UGC 冷启动空白。
