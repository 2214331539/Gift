# 部署运维与数据指标

## 1. 部署目标

Gift 第一版需要部署为真实可用的网站，支持公开访问、用户注册、图片上传、内容审核和基础数据统计。

部署目标：

- 生产环境稳定可访问。
- 数据库和图片有备份。
- 错误可追踪。
- 关键指标可观察。
- 能快速回滚。
- 能支持后续迭代。

## 2. 环境划分

### 2.1 local

用途：

- 本地开发。
- 单元测试。
- 功能调试。

资源：

- 本地 PostgreSQL 或 Docker PostgreSQL。
- 本地对象存储模拟器，或开发 bucket。
- 本地环境变量。

### 2.2 staging

用途：

- 上线前验证。
- 数据库迁移演练。
- 管理后台验收。
- 冷启动内容导入演练。

资源：

- 独立数据库。
- 独立对象存储 bucket。
- 独立域名或子域名。

### 2.3 production

用途：

- 正式用户访问。

资源：

- 生产数据库。
- 生产对象存储。
- CDN。
- 域名 HTTPS。
- 监控和备份。

## 3. 环境变量

建议：

```text
DATABASE_URL=
NEXTAUTH_URL=
NEXTAUTH_SECRET=
S3_ENDPOINT=
S3_REGION=
S3_BUCKET=
S3_ACCESS_KEY_ID=
S3_SECRET_ACCESS_KEY=
CDN_BASE_URL=
REDIS_URL=
SENTRY_DSN=
ANALYTICS_ENDPOINT=
EMAIL_PROVIDER=
EMAIL_FROM=
```

安全要求：

- 不把 `.env` 提交到仓库。
- staging 和 production 密钥分离。
- 管理员密钥定期轮换。
- 对象存储访问密钥最小权限。

## 4. 部署方案

### 4.1 托管平台部署

适合：

- 小团队。
- 快速上线。
- 运维能力有限。

架构：

```text
Vercel / Render / Railway
    ↓
托管 PostgreSQL
    ↓
对象存储
    ↓
CDN
```

优点：

- 部署简单。
- HTTPS 自动。
- 回滚方便。

注意：

- 数据库区域和用户访问区域。
- 图片上传跨域配置。
- 函数执行时间限制。

### 4.2 自建服务器部署

适合：

- 希望控制成本。
- 需要部署在指定地区。
- 熟悉 Docker 运维。

架构：

```text
Nginx
    ↓
Next.js standalone Docker container
    ↓
PostgreSQL
    ↓
对象存储 / 本地 MinIO
```

要求：

- 配置 HTTPS。
- 配置进程守护。
- 配置日志。
- 配置数据库备份。
- 配置安全组。

## 5. CI/CD

建议流水线：

```text
代码提交
    ↓
安装依赖
    ↓
类型检查
    ↓
Lint
    ↓
单元测试
    ↓
Build
    ↓
部署 staging
    ↓
人工验收
    ↓
部署 production
```

上线前必须：

- 运行数据库迁移。
- 备份生产数据库。
- 验证对象存储配置。
- 验证管理员账号。

## 6. 数据库运维

### 6.1 迁移

规则：

- 所有结构变更通过迁移文件。
- 生产迁移前先在 staging 演练。
- 迁移前备份。
- 避免生产高峰执行大表锁表迁移。

### 6.2 备份

建议：

- 每日全量备份。
- 重要上线前手动备份。
- 备份保留至少 14-30 天。
- 定期演练恢复。

### 6.3 数据保留

- 审核日志长期保留。
- 举报记录长期保留。
- 埋点事件可按 6-12 个月归档。
- 软删除内容可定期清理图片资源，但需谨慎。

## 7. 图片与对象存储

### 7.1 Bucket 划分

建议：

```text
gift-dev
gift-staging
gift-production
```

### 7.2 图片路径

```text
uploads/{userId}/{yyyy}/{mm}/{assetId}.webp
thumbnails/{userId}/{yyyy}/{mm}/{assetId}.webp
```

不要使用用户原始文件名作为公开路径。

### 7.3 CDN

要求：

- 图片通过 CDN 访问。
- 配置缓存。
- 支持 WebP。
- 如有隐私或违规图片，隐藏后需要能阻断访问或失效缓存。

## 8. 日志与监控

### 8.1 应用日志

记录：

- API 错误。
- 鉴权失败。
- 上传失败。
- 审核操作。
- 举报处理。
- 数据库慢查询。

避免记录：

- 密码。
- 完整 token。
- 敏感隐私文本。

### 8.2 错误监控

建议接入：

- 前端运行时错误。
- 服务端异常。
- API 5xx。
- 图片上传错误。

### 8.3 健康检查

接口：

```http
GET /api/health
```

返回：

```json
{
  "ok": true,
  "data": {
    "app": "ok",
    "database": "ok",
    "storage": "ok"
  }
}
```

## 9. 性能指标

目标：

| 指标 | MVP 目标 |
| --- | --- |
| 首页 TTFB | < 500ms |
| 列表 API P95 | < 1000ms |
| 详情页 API P95 | < 800ms |
| 图片首屏加载 | 使用缩略图和懒加载 |
| 后台列表查询 | < 1000ms |

优化策略：

- 首页精选内容缓存。
- 标签列表缓存。
- 图片 CDN。
- 数据库索引。
- 列表分页。
- 避免详情页 N+1 查询。

## 10. 安全运维

必须配置：

- HTTPS。
- 安全 Cookie。
- CORS 白名单。
- API 限流。
- 上传大小限制。
- 管理后台权限。
- 数据库最小权限账号。
- 对象存储最小权限。

定期检查：

- 管理员账号。
- 外链滥用。
- 异常注册。
- 批量发布。
- 大量举报。

## 11. 数据指标体系

### 11.1 北极星指标

```text
有效帮助次数
```

组成：

- 收藏礼物。
- 点击“我参考了这个”。
- 发布参考版本。
- 发送感谢。
- 创建礼物计划。

MVP 计算方式：

```text
effective_help_count =
  story_save_count +
  fork_click_count +
  thanks_count
```

P1 增加：

```text
+ published_fork_count
+ gift_plan_created_count
```

### 11.2 内容消费指标

- 首页访问量。
- 搜索次数。
- 搜索后点击率。
- 礼物详情页浏览量。
- 平均浏览时长。
- 收藏率。
- 评论率。
- 相似礼物点击率。

关键公式：

```text
搜索后点击率 = 搜索后进入详情次数 / 搜索次数
收藏率 = 收藏次数 / 详情页访问次数
```

### 11.3 内容贡献指标

- 发布礼物数量。
- 发布入口点击率。
- 发布完成率。
- 发布成功率。
- 审核通过率。
- 平均每篇图片数。
- 平均内容完整度。
- 匿名发布比例。
- 失败礼物投稿数量。

关键公式：

```text
发布完成率 = 提交发布人数 / 进入发布页人数
审核通过率 = 审核通过内容数 / 提交审核内容数
```

### 11.4 社区互动指标

- 很用心点击数。
- 收藏数。
- 评论数。
- 被参考次数。
- 感谢数。
- 举报数。
- 精选内容比例。

### 11.5 留存指标

- 次日留存。
- 7 日留存。
- 30 日留存。
- 节日前复访率。
- 收藏后复访率。
- 创建计划后的复访率。

## 13. 冷启动策略

### 13.1 内容冷启动

上线前准备：

```text
至少 100 条真实案例
理想 300 条真实案例
```

内容分布建议：

| 类型 | 数量 |
| --- | --- |
| 送女朋友 | 50 |
| 送男朋友 | 50 |
| 送父母 | 30 |
| 异地恋 | 30 |
| 低预算 | 30 |
| 失败避坑 | 20 |
| 生日 | 20 |
| 纪念日 | 20 |

注意：

- 不盗用别人的图片和文字。
- 可以来自自己和朋友真实投稿。
- 可以邀请种子用户填写。
- 可以内部整理真实案例。
- 冷启动内容必须人工打标签。

### 13.2 种子用户

可邀请：

- 情侣。
- 大学生。
- 刚毕业的人。
- 喜欢做手工的人。
- 摄影爱好者。
- 有仪式感的人。
- 经常做攻略的人。

投稿入口要轻：

```text
上传一张照片 + 回答 5 个问题
```

不要一开始就要求长文。

### 13.3 社交传播

每篇礼物故事可生成分享卡片：

- 标题。
- 一张礼物图。
- 预算。
- 场景。
- 最打动人的一句话。
- Gift 网站标识。

## 14. 上线后运营检查

每日检查：

- 待审核数量。
- 待处理举报。
- 新增内容质量。
- 搜索无结果关键词。
- 上传失败率。
- API 错误。

每周检查：

- 内容贡献漏斗。
- 高收藏内容。
- 高跳出页面。
- 低预算内容表现。
- 标签使用情况。
- 广告和隐私风险。

## 15. 回滚策略

应用回滚：

- 保留上一版本构建。
- 部署异常时回滚应用。

数据库回滚：

- 避免依赖自动 down migration。
- 生产迁移前备份。
- 复杂迁移使用兼容式发布。

内容回滚：

- 精选内容可快速下线。
- 违规内容可隐藏。
- 标签合并应可审计，必要时通过备份恢复。

## 16. 运维验收标准

- 生产域名 HTTPS 可访问。
- 健康检查通过。
- 数据库备份任务正常。
- 图片上传和 CDN 访问正常。
- 错误监控能收到测试错误。
- 管理后台只允许授权角色访问。
- 数据埋点能记录核心事件。
- 冷启动内容导入并可搜索。
- 出现违规内容时管理员能快速隐藏。
