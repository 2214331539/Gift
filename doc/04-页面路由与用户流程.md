# 04. 页面路由与用户流程

## 1. 用户端路由

| 路由 | 页面 | 登录要求 |
| --- | --- | --- |
| `/` | 首页 | 否 |
| `/discover` | 礼物列表 / 搜索结果 | 否 |
| `/stories/:id` | 礼物详情 | 否 |
| `/stories/new` | 发布礼物 | 是 |
| `/stories/:id/fork` | 参考此礼物发布版本 | 是 |
| `/avoid` | 失败礼物专区 | 否 |
| `/festivals/:slug` | 节日专题 | 否 |
| `/collections` | 我的收藏夹 | 是 |
| `/collections/:id` | 收藏夹详情 | 是 |
| `/plans` | 我的礼物计划 | 是 |
| `/plans/new` | 新建礼物计划 | 是 |
| `/users/:id` | 用户主页 | 否 |
| `/me` | 我的主页 | 是 |
| `/settings/profile` | 资料设置 | 是 |
| `/auth/login` | 登录 | 否 |
| `/auth/register` | 注册 | 否 |

## 4. 礼物卡片结构

礼物卡片必须强调故事和场景，不以商品图和价格为主。

字段：

- 封面图。
- 故事标题。
- 送给谁。
- 场景。
- 预算区间。
- 准备时间。
- 2-4 个核心标签。
- 收藏数、参考数、评论数。
- 匿名标识或作者昵称。

## 5. 发布流程

```text
进入发布页
  -> Step 1 基础信息
  -> Step 2 上传照片
  -> Step 3 礼物故事
  -> Step 4 标签预算
  -> Step 5 隐私设置
  -> 提交审核
  -> 发布成功页
```

状态规则：

- 用户可在任意步骤保存草稿。
- 离开页面前如有未保存内容，提示确认。
- 草稿不进入搜索和公开详情。
- 提交后进入 `pending_review`。
- 审核通过后进入 `published`。

## 6. 找礼物流程

```text
首页输入自然语言需求
  -> 系统解析关键词并进入列表页
  -> 用户使用筛选器缩小范围
  -> 查看礼物详情
  -> 收藏 / 评论 / 点击参考
  -> 实际送礼
  -> 回来发布自己的版本
```

关键体验要求：

- 筛选器在移动端使用抽屉，在桌面端使用左侧或顶部筛选区。
- 搜索无结果时，不要只显示空白；提供热门入口和放宽条件。
- 详情页相似推荐要围绕同场景、同预算、同关系，不按价格高低推荐。

## 7. 参考礼物流程

```text
用户在详情页点击“我也参考了这个”
  -> 进入发布页，自动带入原故事 id
  -> 用户填写自己的改造版本
  -> 审核通过
  -> 原故事详情显示“参考这个礼物的人”
  -> 原作者收到通知
```

## 8. 审核流程

```text
用户提交内容
  -> 自动检查敏感词、链接、图片数量、必填字段
  -> 进入待审核
  -> 管理员审核
  -> 通过 / 驳回 / 下架
  -> 用户收到结果通知
```

## 9. 内容状态

| 状态 | 含义 | 用户可见 |
| --- | --- | --- |
| `draft` | 草稿 | 仅作者 |
| `pending_review` | 待审核 | 仅作者、管理员 |
| `published` | 已发布 | 公开 |
| `rejected` | 审核驳回 | 仅作者、管理员 |
| `hidden` | 作者隐藏 | 仅作者、管理员 |
| `removed` | 平台下架 | 仅管理员 |

## 10. 关键空状态

- 未登录收藏：弹登录框，登录后继续收藏。
- 收藏夹为空：推荐创建第一个收藏夹，并展示精选内容。
- 评论为空：提示“还没有人分享参考建议”。
- 审核列表为空：显示今日审核统计。
- 搜索为空：展示可点击的热门标签和放宽筛选按钮。
