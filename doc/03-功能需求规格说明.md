# 03. 功能需求规格说明

## 1. 功能优先级定义

| 优先级 | 含义 |
| --- | --- |
| P0 | MVP 上线必须实现 |
| P1 | 上线后 1-2 个版本内增强 |
| P2 | 长期能力，暂不阻塞上线 |

## 2. 用户端 P0 功能

### 2.1 首页

功能点：

- 主搜索框，支持自然语言输入。
- 快捷入口：女朋友、男朋友、父母、异地恋、低预算、道歉、纪念日、生日。
- 节日倒计时模块。
- 今日精选礼物故事。
- 低预算高心意模块。
- 失败礼物避坑入口。
- 分享礼物入口。

验收标准：

- 未登录用户可浏览首页内容。
- 搜索后进入礼物列表页，并保留搜索词。
- 首页内容可以由后台配置精选和排序。

### 2.2 礼物列表

功能点：

- 关键词搜索。
- 按关系、场景、预算、兴趣、情绪、礼物类型筛选。
- 排序：最相关、最多收藏、最多被参考、最新、低预算优先、准备时间短优先。
- 礼物卡片展示封面、标题、对象、场景、预算、准备时间、标签、收藏数。

验收标准：

- 多个筛选条件可组合。
- 刷新页面后筛选条件不丢失，使用 URL query 表达。
- 空结果页提供换一组筛选、查看热门故事、发布求助提示。

### 2.3 礼物详情

功能点：

- 图片轮播。
- 标题、作者、匿名标识、发布时间。
- 送给谁、场景、关系阶段、预算、准备时间、礼物类型。
- 为什么送、礼物内容、准备过程、对方反应、避坑提醒、适合/不适合人群。
- 收藏、“很用心”、评论、举报。
- 相似礼物推荐。

验收标准：

- 未登录用户可阅读详情，互动动作引导登录。
- 匿名发布时不展示真实作者昵称和头像。
- 下架、审核中、私密内容不能被普通用户访问。

### 2.4 发布礼物

功能点：

- 分 5 步发布：基础信息、上传照片、填写故事、标签预算、隐私设置。
- 支持至少 1 张图片，最多建议 9 张。
- 支持设置封面图。
- 支持匿名发布、隐藏具体价格、隐藏关系时长。
- 发布后进入待审核状态。

必填字段：

- 标题。
- 送给谁。
- 送礼场景。
- 至少 1 张图片。
- 礼物内容描述。
- 为什么送。
- 对方反应或给后来者建议二选一。
- 至少 1 个标签。

验收标准：

- 表单可保存草稿。
- 发布前显示隐私提醒。
- 发布成功页展示贡献感文案和下一步按钮。

### 2.5 登录注册

功能点：

- 邮箱注册、邮箱登录。
- 密码强度校验。
- 退出登录。
- 登录态持久化。

验收标准：

- 需要互动、发布、收藏、评论时必须登录。
- 登录后返回用户原本操作路径。
- 密码不明文存储。

### 2.6 收藏

功能点：

- 收藏按钮文案为“以后想送”。
- 默认收藏夹。
- 自定义收藏夹。
- 取消收藏。

验收标准：

- 同一用户不能重复收藏同一故事到同一收藏夹。
- 收藏数异步统计，前端乐观更新。

### 2.7 评论

功能点：

- 发布一级评论。
- 回复评论。
- 删除自己的评论。
- 举报评论。
- 管理员隐藏评论。

验收标准：

- 评论默认按时间正序或热度展示。
- 包含敏感词的评论进入待审核或发布失败。
- 被隐藏评论对普通用户不可见。

## 3. 管理端 P0 功能

### 3.1 内容审核

功能点：

- 待审核列表。
- 礼物详情预览。
- 通过、驳回、下架。
- 填写驳回原因。
- 标记精选。

审核维度：

- 隐私泄露。
- 广告导流。
- 盗图搬运。
- 攻击、歧视、低俗。
- 违法或危险行为。
- 标题党。
- 重复内容。

### 3.2 举报处理

功能点：

- 举报列表。
- 按目标类型过滤：故事、评论、用户。
- 处理结果：无效、已处理、需要补充信息。
- 处理动作：隐藏、下架、封禁、忽略。

### 3.3 标签管理

功能点：

- 新增、编辑、禁用标签。
- 设置标签分类。
- 合并同义标签。
- 设置热门标签。
- 查看标签使用次数。

### 3.4 精选内容管理

功能点：

- 首页精选。
- 低预算精选。
- 失败避坑精选。
- 节日专题精选。
- 支持排序和上下架。

## 4. P1 功能
