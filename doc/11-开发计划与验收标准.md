# 开发计划与验收标准

## 1. 交付目标

第一版交付一个真实可用的 Gift Web 网站，至少支持：

- 用户浏览真实礼物故事。
- 用户搜索和筛选礼物故事。
- 用户查看礼物详情。
- 用户注册登录。
- 用户发布礼物故事并上传图片。
- 用户收藏、评论和“很用心”。
- 管理员审核内容、处理举报、管理标签和精选。
- 网站可以部署到生产环境并持续维护。

## 2. 里程碑规划

建议以 6-8 周完成 MVP。若团队只有 1 人，可延长到 8-10 周。

### 2.1 第 0 阶段：项目初始化，2-3 天

目标：

- 建立工程骨架。
- 确定技术栈。
- 建立数据库迁移。
- 配置开发环境。

任务：

- 初始化 Next.js + TypeScript。
- 配置 Tailwind CSS。
- 配置 Prisma 和 PostgreSQL。
- 配置 ESLint、Prettier、测试框架。
- 配置环境变量。
- 建立基础布局和路由。
- 建立 CI 构建检查。

验收：

- 本地能启动 Web 服务。
- 本地能连接数据库。
- 可以执行数据库迁移。
- 代码提交前能运行 lint 和 build。

### 2.2 第 1 阶段：基础数据与内容浏览，1 周

目标：

- 建立礼物故事基础模型。
- 完成首页、列表页、详情页只读链路。

任务：

- 实现 `users`、`gift_stories`、`tags`、`media_assets` 基础表。
- 初始化系统标签。
- 导入冷启动示例内容。
- 实现首页聚合接口。
- 实现礼物列表接口。
- 实现礼物详情接口。
- 实现 StoryCard、StoryGrid、ImageGallery。
- 实现基础 SEO title 和 meta。

验收：

- 首页能展示精选、热门场景、低预算、失败避坑内容。
- 礼物列表能展示公开内容。
- 礼物详情能展示完整故事字段。
- 只展示 `approved + public` 内容。
- 页面在移动端和桌面端可用。

### 2.3 第 2 阶段：搜索、筛选和标签，1 周

目标：

- 用户能通过关键词和标签找到礼物。

任务：

- 实现标签列表 API。
- 实现列表页筛选。
- 实现关键词搜索。
- 实现排序：最相关、最新、最多收藏、低预算。
- 实现 URL query 状态同步。
- 实现无结果状态。
- 实现标签后台基础管理。

验收：

- 搜索“女朋友 生日 300”能返回匹配内容。
- 多个筛选条件能组合。
- 刷新页面后筛选条件保留。
- 无结果时给出可操作建议。
- 标签可由管理员新增、编辑、禁用。

### 2.4 第 3 阶段：用户系统与互动，1 周

目标：

- 支持注册登录、收藏、评论、“很用心”。

任务：

- 接入 Auth.js 或同类鉴权。
- 实现注册、登录、当前用户接口。
- 实现收藏和取消收藏。
- 实现评论列表和发布评论。
- 实现“很用心”互动。
- 实现我的收藏。
- 实现用户主页基础版。

验收：

- 游客可以浏览，但写操作引导登录。
- 登录后能收藏并取消收藏。
- 登录后能评论。
- 同一用户不能重复收藏同一故事。
- 用户主页能展示发布和贡献指标。

### 2.5 第 4 阶段：发布礼物与图片上传，1.5 周

目标：

- 用户可以发布自己的礼物故事。

任务：

- 实现发布 Stepper。
- 实现图片上传预签名接口。
- 实现对象存储配置。
- 实现图片上传、预览、封面选择、删除。
- 实现发布表单校验。
- 实现草稿保存。
- 实现提交审核。
- 实现发布成功页。
- 实现隐私提醒和匿名发布。

验收：

- 用户可以从发布入口完成完整发布流程。
- 至少一张图片才能提交。
- 必填字段缺失时不能提交。
- 发布后状态为 `pending`。
- 匿名发布前台不暴露作者身份。
- 图片上传失败可重试。
- 发布前显示隐私提醒。

### 2.6 第 5 阶段：管理后台与审核，1 周

目标：

- 形成内容审核闭环，防止不合格内容直接公开。

任务：

- 实现后台权限。
- 实现内容审核列表。
- 实现内容审核详情。
- 实现通过、驳回、隐藏。
- 实现举报提交。
- 实现举报后台。
- 实现精选内容管理。
- 实现审核日志。

验收：

- 管理员能查看待审核内容。
- 通过后内容公开展示。
- 驳回后用户能看到原因。
- 隐藏后前台不可见。
- 用户能举报内容和评论。
- 举报能在后台处理。
- 所有审核操作写入日志。

### 2.7 第 6 阶段：质量、部署和上线，1 周

目标：

- 完成生产部署前的质量检查。

任务：

- 补充关键测试。
- 配置生产环境变量。
- 配置数据库备份。
- 配置对象存储和 CDN。
- 配置域名和 HTTPS。
- 配置错误监控。
- 配置基础数据统计。
- 导入 100-300 条冷启动内容。
- 完成上线检查。

验收：

- 生产环境可访问。
- 用户能完成核心路径。
- 管理后台可用。
- 错误监控正常。
- 数据库备份正常。
- 冷启动内容可检索。

## 4. 测试策略

### 4.1 单元测试

覆盖：

- 表单校验。
- 权限判断。
- 内容状态流转。
- 搜索参数解析。
- 标签合并逻辑。
- 内容完整度计算。

### 4.2 集成测试

覆盖：

- 注册登录。
- 发布故事。
- 提交审核。
- 审核通过。
- 搜索公开内容。
- 收藏和取消收藏。
- 评论。
- 举报处理。

### 4.3 E2E 测试

关键路径：

```text
游客首页搜索 → 列表 → 详情
注册用户登录 → 收藏 → 我的收藏
注册用户发布礼物 → 后台审核通过 → 前台可见
注册用户评论 → 举报评论 → 后台处理
管理员管理标签 → 前台筛选可用
```

### 4.4 手工验收

必须手工检查：

- 移动端布局。
- 图片上传体验。
- 发布流程文案。
- 隐私提醒。
- 后台审核效率。
- 首页是否过度导购化。

## 5. 模块验收标准

### 5.1 首页

- 搜索框在首屏明显位置。
- 快捷入口可点击。
- 展示精选、热门场景、低预算、失败避坑。
- 点击卡片进入详情。
- 移动端不出现横向滚动。

### 5.2 礼物列表

- 支持关键词搜索。
- 支持关系、场景、预算、类型筛选。
- 支持排序。
- URL 保留筛选条件。
- 空状态有可操作建议。

### 5.3 礼物详情

- 展示图片、标题、标签、原因、反应、预算、准备时间、适合和不适合。
- 支持收藏、评论、很用心、举报。
- 匿名内容不展示真实作者。
- 相似推荐不包含当前内容。

### 5.4 发布礼物

- 必填字段完整校验。
- 至少上传一张图片。
- 支持匿名发布。
- 支持隐藏具体价格。
- 发布前有隐私提醒。
- 发布后进入待审核。
- 用户可以查看自己的发布状态。

### 5.5 管理后台

- 普通用户无法访问。
- 管理员能筛选待审核内容。
- 审核通过后前台可见。
- 驳回必须填写原因。
- 举报可以处理并记录结果。
- 标签可以新增、编辑、禁用、合并。
- 精选内容可以配置。

### 5.6 安全与隐私

- 写操作必须登录。
- 用户不能编辑他人内容。
- 管理接口必须校验角色。
- 上传文件类型和大小受限制。
- 待审核内容不可公开访问。
- 审核日志完整。

## 6. 上线前检查清单

产品：

- MVP 功能均完成。
- 冷启动内容不少于 100 条。
- 首页内容不导购化。
- 低预算内容有固定入口。
- 失败避坑内容可浏览。

技术：

- 生产环境 build 通过。
- 数据库迁移成功。
- 对象存储可用。
- 图片 CDN 可用。
- 环境变量完整。
- 域名 HTTPS 生效。
- 错误监控可用。
- 数据库备份可用。

安全：

- 管理后台权限正确。
- 上传限制生效。
- XSS 基础防护。
- CSRF 或同等防护。
- 管理操作审计日志。

数据：

- 埋点事件正常。
- 搜索日志可记录。
- 发布漏斗可分析。
- 收藏和有效帮助次数可统计。

## 7. 发布后观察

上线后前 2 周重点看：

- 搜索词。
- 搜索无结果。
- 详情点击率。
- 收藏率。
- 发布入口点击率。
- 发布完成率。
- 审核通过率。
- 举报类型。
- 用户最常用筛选。
- 低预算内容表现。

## 9. 完成定义

一个任务只有在满足以下条件时才算完成：

- 功能按需求可用。
- 错误状态已处理。
- 权限规则正确。
- 移动端可用。
- 数据落库正确。
- 关键路径有测试或手工验收记录。
- 不引入明显性能问题。
- 不破坏产品定位。
