# 管理后台设计

## 1. 后台目标

Gift 的管理后台不是复杂 CRM，而是保证社区内容质量和安全的运营工具。第一版后台重点解决：

- 内容审核。
- 举报处理。
- 标签管理。
- 精选内容管理。
- 基础数据查看。

后台设计应高效、清晰、可追踪。所有审核和管理动作必须写入审计日志。

## 2. 后台角色

| 角色 | 权限 |
| --- | --- |
| 审核员 moderator | 内容审核、评论审核、举报处理、查看标签 |
| 管理员 admin | 审核员全部权限 + 标签管理 + 用户管理 + 精选管理 + 系统配置 |

## 3. 后台导航

```text
仪表盘
内容审核
评论管理
举报处理
标签管理
精选内容
节日专题
用户管理
数据统计
审计日志
系统设置
```

MVP 必须：

- 内容审核
- 举报处理
- 标签管理
- 精选内容
- 审计日志

## 4. 仪表盘

MVP 展示：

- 待审核内容数。
- 待处理举报数。
- 今日新增故事数。
- 今日新增用户数。
- 今日发布通过数。
- 疑似广告内容数。
- 近 7 日发布趋势。

快捷入口：

- 处理待审核内容。
- 处理举报。
- 新增精选内容。
- 管理标签。

## 5. 内容审核

### 5.1 列表字段

| 字段 | 说明 |
| --- | --- |
| 标题 | 礼物故事标题 |
| 作者 | 昵称或匿名 |
| 状态 | pending, approved, rejected, hidden |
| 图片数 | 图片数量 |
| 必填完整度 | 核心字段完整情况 |
| 风险标记 | 隐私、广告、盗图、低俗等 |
| 标签 | 送给谁、场景、预算、类型 |
| 提交时间 | pending 时间 |
| 操作 | 查看、通过、驳回、隐藏 |

筛选：

- 状态。
- 场景。
- 是否失败案例。
- 是否匿名。
- 风险类型。
- 提交时间。
- 作者。

排序：

- 最新提交。
- 风险优先。
- 被举报优先。
- 内容完整度低优先。

### 5.2 审核详情

展示：

- 标题。
- 作者信息。
- 图片预览。
- 送给谁、场景、预算、礼物类型。
- 为什么送。
- 准备过程。
- 对方反应。
- 踩坑提醒。
- 适合 / 不适合。
- 标签。
- 隐私确认状态。
- 系统风险提示。
- 历史审核日志。

操作：

- 通过。
- 驳回。
- 隐藏。
- 标记精选，管理员。
- 编辑标签，管理员。

### 5.3 驳回原因模板

常用模板：

- 图片中包含姓名、地址、电话、订单号等隐私信息，请处理后重新提交。
- 内容更像商品推荐或广告，请补充真实送礼背景和对方反应。
- 礼物故事信息不完整，请补充“为什么送”和“TA 的反应”。
- 图片疑似来自商品详情页，请上传真实照片。
- 内容包含攻击或不适当表达，请修改后提交。

### 5.4 审核规则

- 审核通过后内容变为公开。
- 驳回必须填写原因。
- 隐藏公开内容必须填写原因。
- 审核员不能直接修改正文，除非后续设计“运营代改”权限。
- 所有操作写入 `moderation_logs`。

## 6. 举报处理

### 6.1 举报列表字段

| 字段 | 说明 |
| --- | --- |
| 举报类型 | 隐私、广告、盗图等 |
| 举报目标 | 故事、评论、用户 |
| 举报人 | 用户信息 |
| 目标状态 | 当前是否公开 |
| 举报说明 | 用户填写内容 |
| 举报时间 | 创建时间 |
| 处理状态 | pending, reviewing, resolved, rejected |
| 操作 | 查看、处理 |

### 6.2 处理详情

展示：

- 举报原因。
- 举报说明。
- 目标内容快照。
- 目标作者。
- 目标历史举报次数。
- 相关审核日志。

处理动作：

- 标记无效。
- 隐藏目标内容。
- 驳回并要求作者修改。
- 删除目标内容。
- 限制作者。
- 封禁作者，管理员。

处理结果必须记录：

- 动作。
- 说明。
- 处理人。
- 时间。

## 7. 标签管理

### 7.1 标签列表

字段：

- 名称。
- 类型。
- slug。
- 父标签。
- 主标签。
- 使用次数。
- 是否系统标签。
- 是否启用。
- 排序。

操作：

- 新增。
- 编辑。
- 禁用。
- 合并。
- 设置父标签。
- 设置热门。

### 7.2 标签合并

场景：

```text
异地
远距离恋爱
异地恋
```

合并为：

```text
异地恋
```

合并操作需要：

1. 选择源标签。
2. 选择目标主标签。
3. 预览影响内容数量。
4. 确认合并。
5. 更新 `canonical_id` 或迁移关联表。
6. 写入审计日志。

### 7.3 标签质量规则

需要处理：

- 重复标签。
- 错别字标签。
- 过窄标签。
- 广告标签。
- 与已有标签含义相近的标签。

## 8. 精选内容管理

### 8.1 精选位

MVP 精选位：

- 首页今日精选。
- 低预算高心意。
- 失败礼物避坑。
- 节日倒计时推荐。
- 最新分享，可自动。

字段：

- 精选位。
- 礼物故事。
- 展示标题覆盖。
- 排序。
- 开始时间。
- 结束时间。
- 是否启用。

### 8.2 精选标准

优先：

- 真实度高。
- 有故事感。
- 有参考价值。
- 预算友好。
- 不是单纯贵重礼物。
- 评论区反馈好。
- 被多人参考。
- 有后续更新。

避免：

- 明显广告。
- 高价炫耀。
- 信息不完整。
- 隐私风险。

## 9. 评论管理

MVP 可简化为举报驱动。P1 建议增加评论管理。

功能：

- 查看最新评论。
- 筛选被举报评论。
- 隐藏评论。
- 删除评论。
- 限制用户评论。

重点检查：

- 恶意嘲讽。
- 广告链接。
- 隐私索取。
- 人身攻击。

## 10. 用户管理

P1 功能：

- 用户列表。
- 查看用户发布内容。
- 查看举报记录。
- 限制发布。
- 限制评论。
- 封禁账号。
- 解除限制。

MVP 至少需要管理员账号管理方式，可以通过数据库初始化或简化后台实现。

## 11. 数据统计

MVP 后台基础指标：

- 总用户数。
- 总礼物故事数。
- 审核通过率。
- 发布成功率。
- 搜索次数。
- 收藏次数。
- 评论次数。
- 举报次数。
- 有效帮助次数。

P1 增加：

- 搜索无结果关键词。
- 高收藏标签。
- 高转化场景。
- 内容贡献漏斗。
- 发布中断步骤。

## 12. 审计日志

记录内容：

- 操作人。
- 操作目标。
- 操作类型。
- 操作前状态。
- 操作后状态。
- 操作原因。
- 操作时间。

后台需要提供审计日志查询：

- 按操作人。
- 按目标类型。
- 按时间。
- 按操作类型。

## 13. 后台安全

要求：

- 后台路由必须登录。
- 后台接口必须校验角色。
- 管理员危险操作二次确认。
- 管理员账号建议开启二次验证。
- 后台操作限流。
- 审计日志不可被普通管理员删除。

## 14. 后台验收标准

- 审核员能看到待审核内容并完成通过/驳回。
- 驳回必须填写原因，用户能看到原因。
- 管理员能处理举报并记录结果。
- 管理员能新增、编辑、禁用、合并标签。
- 管理员能配置首页精选内容。
- 所有后台操作写入审计日志。
- 普通用户无法访问后台页面和接口。
