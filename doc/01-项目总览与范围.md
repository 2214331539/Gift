# 01. 项目总览与范围

## 1. 产品定位

Gift 是一个真实礼物灵感社区。用户可以在生日、纪念日、节日、道歉、异地见面、毕业、入职等场景下，搜索别人真实送过的礼物故事，并发布自己的送礼经验。

产品核心不是“商品推荐”，而是“在具体关系、预算、场景和情绪下，什么礼物能让对方感受到被理解”。

## 2. 核心价值

- 真实案例：展示用户真实送过的礼物、准备过程和对方反应。
- 场景匹配：围绕关系、节日、预算、兴趣、情绪、准备时间筛选。
- 可复用灵感：允许用户收藏、参考、改造并再次分享。
- 失败避坑：展示失败礼物和注意事项，增强内容可信度。
- 用心文化：弱化价格攀比，强调观察、陪伴、纪念意义和适配程度。

## 3. 目标用户

| 用户类型 | 核心诉求 | 关键功能 |
| --- | --- | --- |
| 送礼焦虑型用户 | 快速找到合适灵感 | 搜索、筛选、推荐、收藏 |
| 分享型用户 | 记录并帮助他人 | 引导式发布、匿名、正向反馈 |
| 灵感浏览型用户 | 长期浏览和收藏创意 | 首页精选、标签浏览、收藏夹 |
| 长期计划型用户 | 管理未来送礼计划 | 礼物计划、节日提醒、备选清单 |

## 4. MVP 范围

### 4.1 P0 必做

- 首页：主搜索、热门场景、精选故事、低预算入口、失败避坑入口。
- 礼物列表：关键词搜索、多维筛选、排序、分页。
- 礼物详情：图片、礼物故事、预算、准备时间、适合人群、避坑提醒、评论、相似推荐。
- 发布礼物：分步表单、图片上传、标签选择、匿名发布、隐私提示。
- 用户系统：注册、登录、退出、个人主页基础版。
- 收藏功能：收藏到“以后想送”或自定义收藏夹。
- 评论功能：一级评论、回复、删除本人评论、举报评论。
- 内容审核：新内容待审核、通过、驳回、下架。
- 标签系统：关系、场景、预算、兴趣、情绪、礼物类型、内容质量。
- 管理后台：内容审核、举报处理、标签管理、精选内容管理。

### 4.2 P1 增强

- “我也参考了这个”礼物版本关系。
- 礼物计划和节日提醒。
- 感谢机制。
- 失败礼物专区完善。
- 内容完整度评分。
- 用户贡献指标和个人主页增强。
- 节日专题页。

### 4.3 P2 长期

- AI 礼物助手。
- 语义搜索。
- 个性化推荐。
- 灵感树可视化。
- 图片隐私自动识别。
- 多语言支持。
- 移动端 App。

## 5. 明确不做

MVP 不做以下能力：

- 商品交易、购物车、支付。
- 商家入驻、广告投放、电商导购分佣。
- 私信、关注、复杂社交关系链。
- 视频发布。
- 排行榜和复杂积分系统。
- 高复杂度 AI 生成内容。

## 6. 北极星指标

北极星指标：有效帮助次数。

有效帮助由以下行为累计：

- 收藏礼物故事。
- 点击“我也参考了这个”。
- 发布参考改造版本。
- 对原作者发送感谢。
- 创建礼物计划。

## 7. 核心业务闭环

```text
用户遇到送礼需求
  -> 搜索 / 浏览 Gift
  -> 查看真实礼物故事
  -> 收藏 / 参考 / 评论
  -> 实际送出礼物
  -> 回来分享自己的版本
  -> 其他用户继续参考
```

## 8. 成功标准

- 用户能在 3 次筛选内找到至少 1 个可参考案例。
- 发布流程在 5 分钟内可完成一篇基础礼物故事。
- 首页和列表页不呈现明显导购站气质。
- 礼物详情页必须能回答“为什么送、怎么准备、对方反应、有哪些坑”。
- 后台能及时处理广告、盗图、隐私泄露和低质内容。
