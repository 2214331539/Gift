# 07. 搜索推荐与标签体系

## 1. 标签体系目标

Gift 的标签是搜索、筛选、推荐和内容组织的基础。标签不能只做“生日”“女朋友”这种浅层分类，还要覆盖情绪、兴趣、礼物类型和内容质量。

## 2. 标签分类

| 分类 | 示例 | 用途 |
| --- | --- | --- |
| 关系标签 | 女朋友、男朋友、父母、朋友、同事 | 判断送给谁 |
| 场景标签 | 生日、纪念日、异地恋、道歉、毕业 | 判断为什么送 |
| 预算标签 | 0 元、50 元以内、300 元以内、不花钱但用心 | 控制预算预期 |
| 兴趣标签 | 摄影、游戏、音乐、健身、阅读、猫狗 | 匹配对方偏好 |
| 情绪标签 | 安慰、鼓励、感谢、惊喜、表达认真 | 匹配用户表达目的 |
| 礼物类型 | DIY、手写信、体验、数码、美食、陪伴 | 匹配礼物形态 |
| 内容质量 | 真实照片、有后续反馈、低预算高心意、容易翻车 | 排序和推荐加权 |

## 3. 标签治理规则

- 标签由后台维护，普通用户发布时只能选择已有标签，P1 可开放用户建议标签。
- 同义标签必须合并，例如“异地”“异地恋”“远距离恋爱”统一到“异地恋”。
- 避免强化刻板印象，例如不能默认“男生都喜欢数码”。
- 热门标签由运营配置和使用次数共同决定。
- 低质量、重复、歧义标签应禁用，不直接删除历史数据。

## 4. 搜索输入类型

Gift 需要支持两类输入：

1. 结构化筛选：关系、场景、预算、兴趣、情绪、类型。
2. 自然语言：例如“男朋友最近压力大，预算 300，想让他放松”。

MVP 的自然语言处理先使用规则解析：

- 识别预算数字和区间。
- 匹配关系词。
- 匹配场景词。
- 匹配情绪词。
- 将剩余文本作为全文关键词。

## 5. 搜索召回策略

MVP 召回顺序：

```text
强标签匹配
  + 全文搜索
  + 标题关键词
  + 内容关键词
  + 热门精选补充
```

如果结果过少：

- 放宽兴趣标签。
- 放宽预算上限。
- 保留关系和场景。
- 插入运营精选内容。

## 6. 排序权重

建议计算 `rank_score`：

| 因子 | 权重建议 | 说明 |
| --- | --- | --- |
| 关系匹配 | 25 | 送给谁最重要 |
| 场景匹配 | 20 | 生日、道歉、毕业等强场景 |
| 预算匹配 | 15 | 预算不符会明显降低可用性 |
| 情绪匹配 | 15 | 安慰、感谢、惊喜等核心价值 |
| 兴趣匹配 | 10 | 个性化辅助 |
| 内容质量 | 10 | 真实照片、反应、建议、完整度 |
| 互动反馈 | 5 | 收藏、评论、被参考 |

排序原则：

- 适配度优先于热度。
- 不按价格高低制造攀比。
- 新内容需要一定曝光，避免冷启动后永远无流量。
- 失败避坑内容在相关查询中应有曝光。

## 7. 推荐场景

### 7.1 首页推荐

混合展示：

- 热门真实故事。
- 最新发布。
- 低预算高心意。
- 失败避坑。
- 节日相关。
- 被多人参考的故事。

### 7.2 详情页相似推荐

优先级：

```text
同场景 > 同关系 > 同预算 > 同情绪 > 同礼物类型 > 同兴趣
```

详情页还应展示：

- 参考这个礼物的人。
- 同源灵感的其他版本。
- 失败避坑或替代方案。

### 7.3 搜索结果推荐

搜索结果中可以插入：

- “更低预算也可参考”。
- “需要更快准备？”。
- “同场景失败避坑”。

## 8. 内容质量评分

`quality_score` 满分 100：

| 项 | 分值 |
| --- | --- |
| 有真实图片 | 20 |
| 写清为什么送 | 20 |
| 写清对方反应 | 20 |
| 写清预算和准备时间 | 15 |
| 写了避坑或建议 | 15 |
| 标签完整 | 10 |

内容完整度用于：

- 发布页提示。
- 搜索排序加权。
- 后台精选判断。

## 9. 搜索指标

- 搜索次数。
- 搜索后点击率。
- 零结果率。
- 筛选使用率。
- 搜索后收藏率。
- 搜索后发布参考版本率。
- 热门搜索词。
- 低相关反馈次数。
