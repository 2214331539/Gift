# 05. 数据接口设计

## 1. API 约定

基础路径：`/api/v1`

请求格式：`application/json`

认证方式：

- API 请求通过 `Authorization: Bearer <access_token>` 认证。
- Access Token 有效期 15 分钟，过期后通过 Refresh Token 静默刷新。
- Refresh Token 存储于 HttpOnly Cookie，有效期 7 天。
- 管理端接口必须校验角色（admin / moderator / editor）。
- 图片上传使用后端签发的 S3 预签名 URL，前端直传。

统一响应：

```json
{
  "data": {},
  "meta": {},
  "error": null
}
```

统一错误：

```json
{
  "data": null,
  "meta": {},
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "请求参数不合法",
    "details": []
  }
}
```

分页参数：

| 参数 | 说明 |
| --- | --- |
| `page` | 页码，从 1 开始 |
| `pageSize` | 每页数量，默认 20，最大 50 |
| `cursor` | 游标分页场景使用（评论、通知等时间线数据） |

```json
{
  "email": "user@example.com",
  "password": "StrongP@ss1",
  "nickname": "送礼达人"
```

```json
{
  "email": "user@example.com",
  "password": "password"

```

```json
  }

```

```json

  "data": {
```


## 4. 礼物故事接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/stories` | 礼物列表、搜索、筛选 |
| `GET` | `/stories/:id` | 礼物详情 |
| `POST` | `/stories` | 创建礼物故事（草稿） |
| `PATCH` | `/stories/:id` | 更新草稿或被驳回内容 |
| `DELETE` | `/stories/:id` | 作者删除或隐藏 |
| `POST` | `/stories/:id/submit` | 提交审核 |
| `POST` | `/stories/:id/view` | 记录浏览 |

列表查询参数（支持多选，多值用逗号分隔）：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `q` | string | 自然语言或关键词 |
| `recipientType` | string | 送给谁，多选逗号分隔：`girlfriend,boyfriend` |
| `occasion` | string | 场景，多选逗号分隔：`birthday,anniversary` |
| `budgetRange` | string | 预算，多选逗号分隔：`0_50,50_100` |
| `giftType` | string | 礼物类型，多选逗号分隔 |
| `emotion` | string | 情绪，多选逗号分隔 |
| `interest` | string | 兴趣，多选逗号分隔 |
| `preparationTime` | string | 准备时间 |
| `isFailure` | boolean | 是否失败礼物 |
| `sort` | string | 排序：`relevant` / `most_saved` / `most_forked` / `newest` / `low_budget` / `quick_prep` |
| `page` | number | 页码 |
| `pageSize` | number | 每页数量 |

## 5. 图片接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/media/upload-token` | 获取 S3 预签名上传 URL |
| `POST` | `/media/complete` | 上传完成回调 |
| `PATCH` | `/media/:id` | 更新 alt、裁剪信息 |
| `DELETE` | `/media/:id` | 删除未使用图片 |

获取上传凭证请求：

```json
{
  "fileName": "gift.jpg",
  "mimeType": "image/jpeg",
  "size": 1200344
}
```

获取上传凭证响应：

```json
{
  "data": {
    "uploadUrl": "https://s3.example.com/presigned...",
    "objectKey": "stories/2026/05/abc123.jpg",
    "expiresIn": 600,
    "mediaId": "uuid"
  }
}
```

上传约束（后端校验）：

| 约束 | 值 |
| --- | --- |
| 单文件最大 | 10 MB |
| 允许 MIME | image/jpeg, image/png, image/webp, image/heic |
| 凭证有效期 | 10 分钟 |
| 单篇故事最多 | 9 张 |
| 每日每用户上限 | 100 张 |

上传完成请求：

```json
{
  "mediaId": "uuid",
  "objectKey": "stories/2026/05/abc123.jpg",
  "width": 1200,
  "height": 900
}
```

## 13. 错误码

| code | HTTP 状态码 | 说明 |
| --- | --- | --- |
| `UNAUTHORIZED` | 401 | 未登录或 Token 无效 |
| `TOKEN_EXPIRED` | 401 | Access Token 过期，前端应尝试刷新 |
| `REFRESH_TOKEN_INVALID` | 401 | Refresh Token 无效或已吊销，需重新登录 |
| `FORBIDDEN` | 403 | 无权限 |
| `VALIDATION_ERROR` | 422 | 参数校验失败 |
| `NOT_FOUND` | 404 | 资源不存在 |
| `CONFLICT` | 409 | 重复操作（如重复收藏） |
| `RATE_LIMITED` | 429 | 请求过快 |
| `CONTENT_UNDER_REVIEW` | 403 | 内容审核中，不可公开访问 |
| `CONTENT_REJECTED` | 403 | 内容被驳回 |
| `UPLOAD_FAILED` | 500 | 文件上传失败 |
| `UPLOAD_SIZE_EXCEEDED` | 413 | 文件超过大小限制 |
| `UPLOAD_TYPE_NOT_ALLOWED` | 415 | 文件类型不允许 |
| `EMAIL_NOT_VERIFIED` | 403 | 邮箱未验证，不可发布内容 |
| `INTERNAL_ERROR` | 500 | 服务器错误 |
