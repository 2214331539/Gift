# 项目范围与商业目标

## 1. 项目背景

送礼是一个高频但高压力的行为。用户在生日、纪念日、节日、道歉、异地见面、毕业、入职、考试结束、见家长等场景下，常常不是缺少商品选择，而是不知道怎样表达理解、陪伴、纪念和用心。

现有渠道各有明显问题：

| 渠道 | 主要问题 |
| --- | --- |
| 电商平台 | 商品很多，但缺少真实故事、关系语境和对方反应 |
| 小红书/社交平台 | 内容分散、广告多、搜索不精准 |
| 知乎/论坛 | 文字多，图片、结构化标签和筛选能力弱 |
| 朋友推荐 | 覆盖面有限，未必适合自己的对象 |
| AI 推荐 | 反馈快，但缺少真实案例支撑 |
| 礼物榜单网站 | 容易模板化，无法处理具体关系和情绪 |

Gift 的机会是把“真实案例 + 场景标签 + 情绪故事 + 可复用灵感”集中沉淀成一个可搜索、可收藏、可贡献、可改造的网站。

## 2. 产品定位

Gift 是一个 UGC 礼物灵感社区，也是一个开源礼物合集网站。

它不是商品推荐站，不以导购转化为第一目标。它服务的是用户真实问题：

> 在这个关系、这个场景、这个预算、这个情绪下，什么样的礼物会让对方感受到被理解？

核心内容不是商品本身，而是：

- 真实送礼案例
- 礼物背后的原因
- 对方收到后的反应
- 适合什么样的人
- 是否值得参考
- 有哪些坑
- 其他人如何基于这个灵感做了改造

## 3. 目标用户

### 3.1 送礼焦虑型用户

特征：

- 节日前几天才开始焦虑
- 不知道送什么
- 怕送错、怕不用心
- 预算可能有限
- 需要快速获得灵感

典型问题：

- 女朋友生日送什么比较用心？
- 男朋友真的会喜欢什么礼物？
- 预算 300 元以内送什么不廉价？
- 异地恋见面送什么比较有纪念意义？
- 七夕不想送花和口红，还有什么选择？

产品要满足：

- 搜索精准
- 分类清楚
- 案例真实
- 快速收藏
- 有避坑信息
- 有类似案例推荐

### 3.2 分享型用户

特征：

- 刚送完一次满意的礼物
- 想记录过程
- 想帮助别人
- 希望得到认可
- 可能不想暴露身份

产品要满足：

- 发布简单
- 支持匿名
- 有引导式模板
- 上传照片方便
- 发布后有正向反馈
- 内容能被别人收藏、感谢、参考

### 3.3 灵感浏览型用户

特征：

- 暂时不一定要送礼
- 喜欢看别人如何表达心意
- 喜欢收藏创意
- 会长期浏览

产品要满足：

- 内容有故事感
- 首页推荐有趣
- 卡片足够吸引人
- 支持收藏夹
- 支持按情绪、关系和场景浏览

### 3.4 长期计划型用户

特征：

- 重视仪式感
- 会提前准备生日、纪念日、节日
- 喜欢建立清单
- 可能每年都会回来

产品要满足：

- 礼物收藏夹
- 礼物计划
- 节日提醒
- 对象档案
- 已送礼物记录
- 未送礼物灵感库

## 4. 商业目标

### 4.1 第一阶段目标

第一阶段的目标不是盈利，而是验证真实需求和内容闭环。

业务目标：

1. 建立一批可搜索、可浏览、可参考的高质量礼物案例。
2. 让用户愿意通过搜索、标签和筛选找礼物。
3. 让用户愿意收藏礼物，为未来送礼做准备。
4. 让用户愿意发布自己的送礼故事。
5. 防止产品变成广告站、电商导购站或炫富社区。

### 4.2 北极星指标

建议北极星指标：

> 有效帮助次数

有效帮助由以下行为组成：

- 收藏礼物
- 点击“我参考了这个”
- 发布参考版本
- 发送感谢
- 创建礼物计划

该指标比 PV、点赞数和热榜排名更符合 Gift 的产品价值。

### 4.3 阶段性指标

内容消费指标：

- 首页访问量
- 搜索次数
- 搜索后点击率
- 礼物详情页浏览量
- 平均浏览时长
- 收藏率
- 评论率
- 相似礼物点击率

内容贡献指标：

- 发布礼物数量
- 发布成功率
- 发布完成率
- 平均每篇图片数
- 平均内容完整度
- 匿名发布比例
- 失败礼物投稿数量

社区互动指标：

- “很用心”点击数
- 收藏数
- 评论数
- 被参考次数
- 感谢数
- 举报数
- 精选内容比例

留存指标：

- 次日留存
- 7 日留存
- 30 日留存
- 节日前复访率
- 收藏后复访率
- 创建计划后的复访率

## 5. MVP 范围

### 5.1 MVP 要验证的问题

MVP 只验证三件事：

1. 用户是否愿意来看真实礼物案例。
2. 用户是否能通过标签和搜索找到灵感。
3. 用户是否愿意上传自己的礼物故事。

### 5.2 MVP 必须包含

用户端：

- 首页
- 礼物列表页
- 礼物详情页
- 发布礼物页
- 搜索筛选
- 收藏
- 评论
- 登录注册
- 个人主页基础版

管理端：

- 内容审核基础版
- 举报处理基础版
- 标签管理基础版
- 精选内容管理基础版

平台基础能力：

- 图片上传
- 匿名发布
- 隐私提醒
- 内容状态管理
- 基础数据统计

### 5.3 MVP 暂不包含

- 复杂 AI 推荐
- 私信
- 关注系统
- 电商链接体系
- 视频
- 复杂积分
- 排行榜
- 移动端 App
- 礼物灵感树可视化

## 6. 业务边界

### 6.1 平台内容边界

允许：

- 用户真实送礼故事
- 礼物准备过程
- 礼物照片
- 对方反应
- 预算范围
- 适合和不适合场景
- 踩坑经验
- 参考和改造版本

谨慎允许：

- 购买来源
- 商品名称
- 价格区间
- 外部链接

第一版建议对外链严格限制，避免导购化。

不允许：

- 明显广告营销
- 盗图搬运
- 虚假测评
- 隐私泄露
- 人身攻击、低俗、歧视或违法内容
- 批量重复发布

### 6.2 互动边界

鼓励：

- 收藏
- 评论
- “很用心”
- 感谢
- 我参考了这个
- 举报问题内容

不鼓励第一版做：

- 粉丝排名
- 财富展示
- 送礼价格攀比
- 过重社交关系链

## 7. 差异化原则

Gift 的差异化来自以下六点：

1. 真实故事：用户看到的是别人真实送过的礼物，而不是营销榜单。
2. 场景匹配：用户根据关系、预算、节日、情绪、兴趣找到更适合自己的案例。
3. 对方反应：礼物送出去后的真实反馈比商品参数更重要。
4. 失败避坑：平台不仅展示成功礼物，也展示失败礼物。
5. 开源改造：用户可以参考、改造、再分享，形成礼物灵感树。
6. 用心文化：平台氛围强调“用心”，而不是“贵”。

## 8. 成功标准

MVP 上线 4-8 周内，建议用以下标准判断是否继续投入：

| 维度 | 建议标准 |
| --- | --- |
| 内容消费 | 搜索后详情点击率达到 20% 以上 |
| 内容价值 | 详情页收藏率达到 5% 以上 |
| 内容贡献 | 发布入口访问到发布成功转化率达到 8% 以上 |
| 内容质量 | 人工审核通过率达到 70% 以上 |
| 冷启动 | 上线前至少准备 100 条，理想 300 条高质量案例 |
| 社区健康 | 广告、盗图、隐私类举报可被 24 小时内处理 |
| 留存 | 收藏用户 7 日内复访率明显高于未收藏用户 |

## 9. 产品底线

以下情况一旦出现，需要优先处理：

- 首页内容明显导购化
- 高价礼物获得过度曝光
- 用户发布意愿低于预期
- 搜索结果无法匹配场景和情绪
- 图片或文字暴露隐私
- 商家伪装普通用户发内容

所有后续增长和变现方式都不能破坏“真实礼物故事社区”的核心定位。
