# 10. 迭代计划与验收标准

## 1. 里程碑计划

### M0：项目初始化

目标：

- 搭建前后端项目结构。
- 接入数据库、ORM、Redis、对象存储。
- 完成基础 CI。

交付物：

- 可运行的 Web 和 API 服务。
- 数据库 migration 基础表。
- 登录注册最小闭环。

### M1：内容浏览闭环

目标：

- 首页、列表、详情可用。
- 种子数据可导入。
- 基础搜索筛选可用。

交付物：

- 首页模块。
- 礼物列表和筛选。
- 礼物详情。
- 标签数据。

### M2：内容发布闭环

目标：

- 用户可以发布礼物故事。
- 支持图片上传、草稿、提交审核。

交付物：

- 分步发布页。
- 图片上传。
- 草稿保存。
- 提交审核。

### M3：互动闭环

目标：

- 用户可以收藏、评论、举报。

交付物：

- 收藏夹。
- 评论区。
- 举报入口。
- 互动统计。

### M4：管理后台

目标：

- 管理员可以审核内容、处理举报、管理标签和精选。

交付物：

- 审核列表。
- 审核详情。
- 举报处理。
- 标签管理。
- 精选位管理。

### M5：MVP 上线准备

目标：

- 完成测试、性能优化、内容冷启动和上线配置。

交付物：

- 100-300 条种子内容。
- 上线清单完成。
- 监控和备份可用。
- 预发布验收通过。

## 2. 建议排期

| 周期 | 工作 |
| --- | --- |
| 第 1 周 | 项目初始化、设计系统、数据库基础表、登录注册 |
| 第 2 周 | 首页、列表、详情、标签和搜索 |
| 第 3 周 | 发布流程、图片上传、草稿、提交审核 |
| 第 4 周 | 收藏、评论、举报、个人主页基础版 |
| 第 5 周 | 管理后台、审核、标签、精选 |
| 第 6 周 | 联调、测试、性能、安全、内容导入、上线 |

## 3. MVP 验收标准

### 3.1 用户浏览

- 游客可以打开首页、搜索、筛选、查看详情。
- 列表页筛选条件可通过 URL 复现。
- 详情页信息完整，不像商品导购页。
- 相似推荐不为空。

### 3.2 用户发布

- 登录用户可以完成 5 步发布流程。
- 至少上传 1 张图片。
- 可匿名发布。
- 发布后内容进入待审核。
- 被驳回内容可编辑后重新提交。

### 3.3 用户互动

- 登录用户可以收藏和取消收藏。
- 登录用户可以评论、删除自己的评论。
- 用户可以举报故事和评论。
- 未登录用户触发互动时进入登录流程，登录后返回原操作。

### 3.4 管理后台

- 审核员可以查看待审核内容。
- 审核员可以通过、驳回、下架内容。
- 管理员可以新增、编辑、合并标签。
- 运营可以配置首页精选。
- 举报可以被处理并记录日志。

### 3.5 安全隐私

- 普通用户无法访问管理端。
- 匿名内容前台不展示真实作者。
- 图片上传限制类型和大小。
- 新内容默认不直接公开。
- 审核日志可追溯。

## 4. 测试用例重点

| 模块 | 用例 |
| --- | --- |
| Auth | 注册、登录、退出、错误密码、未登录拦截 |
| Story | 草稿、提交、审核通过、驳回、下架 |
| Search | 多筛选组合、空结果、排序、分页 |
| Media | 上传成功、超大文件、错误类型、删除未使用图片 |
| Collection | 收藏、取消、重复收藏、自定义收藏夹 |
| Comment | 发布、回复、删除、敏感词、举报 |
| Admin | 权限拦截、审核、标签合并、精选排序 |

## 5. 内容冷启动计划

上线前准备：

- 50 条送女朋友。
- 50 条送男朋友。
- 30 条送父母。
- 30 条异地恋。
- 30 条低预算。
- 20 条失败避坑。
- 20 条生日。
- 20 条纪念日。

每条种子内容至少包含：

- 标题。
- 送给谁。
- 场景。
- 预算区间。
- 准备时间。
- 至少 1 张图片或可替代配图。
- 为什么送。
- 对方反应。
- 给后来者建议。
- 3 个以上标签。

## 6. 上线后指标

第一阶段观察：

- 首页访问量。
- 搜索次数。
- 搜索后点击率。
- 零结果率。
- 详情页停留时长。
- 收藏率。
- 发布完成率。
- 审核通过率。
- 举报率。

判断 MVP 是否成立：

- 用户不只是浏览首页，而是会进入详情页阅读。
- 搜索后有收藏或继续浏览行为。
- 有用户愿意发布自己的礼物故事。
- 低预算和失败避坑内容有稳定浏览。

## 7. 后续迭代触发条件

| 触发条件 | 下一步 |
| --- | --- |
| 搜索零结果率高 | 优先升级搜索和标签解析 |
| 浏览多但发布少 | 优化发布流程和匿名信任 |
| 收藏多但复访少 | 做礼物计划和节日提醒 |
| 广告举报增加 | 加强反广告审核和新用户限制 |
| 同类内容增多 | 做“我也参考了这个”和版本树 |
