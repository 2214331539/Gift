# 09. 部署运维与开发规范

## 1. 仓库结构建议

```text
Gift
  apps
    web
    api
  packages
    shared
    config
  prisma
    schema.prisma
    migrations
    seed
  doc
  docker
  scripts
```

## 2. 本地开发环境

依赖：

- Node.js LTS。
- pnpm。
- Docker Desktop。
- PostgreSQL。
- Redis。
- S3 兼容对象存储，可本地使用 MinIO。

本地服务：

```text
web: http://localhost:3000
api: http://localhost:4000
postgres: localhost:5432
redis: localhost:6379
minio: localhost:9000
```

## 4. 分支规范

| 分支 | 用途 |
| --- | --- |
| `main` | 生产稳定分支 |
| `develop` | 日常集成 |
| `feature/*` | 功能开发 |
| `fix/*` | Bug 修复 |
| `release/*` | 发布准备 |

提交信息建议：

```text
feat: add story publish flow
fix: prevent duplicate collection item
docs: add api design
refactor: split moderation module
test: add story search tests
```

## 5. 代码规范

- TypeScript 开启严格模式。
- API 入参统一用 Zod 或 DTO 校验。
- 数据库写入必须走 service 层，不在 controller 中直接操作 ORM。
- 前端请求统一封装，处理 401、403、422。
- 组件按 feature 组织，通用组件放 `components`。
- 不在前端硬编码权限，后端必须二次校验。

## 6. 测试策略

| 类型 | 范围 |
| --- | --- |
| 单元测试 | 标签解析、内容完整度评分、权限判断 |
| 集成测试 | API、数据库事务、审核状态流转 |
| E2E 测试 | 注册登录、发布礼物、搜索收藏、审核通过 |
| 视觉检查 | 首页、列表、详情、发布页、管理后台 |

P0 上线前必须覆盖：

- 用户注册登录。
- 发布礼物并进入待审核。
- 管理员审核通过。
- 公开列表可搜到已发布故事。
- 收藏和取消收藏。
- 评论和删除评论。
- 举报并在后台处理。

## 7. CI 流程

每次 PR 执行：

```text
install
lint
typecheck
unit test
build
database migration check
```

合并到 `main` 后：

```text
build docker image
push image
deploy staging
run smoke test
manual approve
deploy production
```

## 8. 部署拓扑

MVP 推荐：

```text
Nginx
  -> web container
  -> api container
api container
  -> postgres
  -> redis
  -> object storage
worker container
  -> redis queue
```

## 9. 数据库迁移

- 所有 schema 变更必须生成 migration。
- 禁止直接在线上手改表结构。
- 破坏性迁移必须先备份。
- 字段删除分两步：先停止写入和读取，再删除字段。

## 10. 日志与监控

必须监控：

- API 错误率。
- P95 响应时间。
- 数据库连接数。
- Redis 队列堆积。
- 图片上传失败率。
- 审核待处理数量。
- 登录失败次数。
- 举报数量突增。

关键业务日志：

- 发布提交。
- 审核通过/驳回。
- 搜索无结果。
- 图片上传失败。
- 举报处理。

## 11. 备份策略

- PostgreSQL 每日全量备份，保留至少 14 天。
- 重要发布前手动备份。
- 对象存储开启版本或定期备份。
- 定期演练恢复流程。

## 12. 上线检查

- HTTPS 正常。
- robots.txt 和 sitemap.xml 正常。
- 管理后台仅授权角色可访问。
- 图片上传大小、类型限制生效。
- 新发布内容默认待审核。
- 邮件发送正常。
- 数据库备份任务已开启。
- 错误日志和告警可用。
