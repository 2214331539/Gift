# 06. 数据库设计

数据库建议使用 PostgreSQL。主键统一使用 `uuid`，时间字段统一使用 `timestamptz`。所有业务表保留 `created_at`、`updated_at`，需要软删除的表保留 `deleted_at`。

## 1. 核心实体关系

```text
users 1----n gift_stories
gift_stories n----n tags
gift_stories 1----n story_images
users 1----n collections 1----n collection_items
gift_stories 1----n comments
gift_stories 1----n reports
gift_stories 1----n gift_forks
users 1----n gift_plans
```

## 16. 数据一致性规则

- 故事被下架后，不从收藏夹删除，但普通用户访问时显示不可见。
- 用户匿名发布只影响前台展示，不影响后台追踪和风控。
- 删除标签前必须先迁移或清理 `story_tags`。
- 收藏数、评论数、参考数使用冗余字段，异步校准。
- 审核日志不可物理删除。
