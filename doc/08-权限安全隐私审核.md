# 08. 权限安全隐私审核

## 1. 角色权限

| 角色 | 权限 |
| --- | --- |
| 游客 | 浏览公开内容、搜索、查看详情 |
| 普通用户 | 发布、收藏、评论、举报、创建礼物计划 |
| editor | 管理精选内容、节日专题 |
| moderator | 审核内容、处理举报、下架评论和故事 |
| admin | 全部权限、用户管理、标签管理、系统配置 |

## 2. 鉴权规则

- 发布、收藏、评论、举报、礼物计划必须登录。
- 邮箱未验证的用户可以浏览和收藏，但不能发布内容和评论。
- 管理端接口必须校验角色和后台访问权限。
- 认证使用 JWT：Access Token（15 分钟）通过 Authorization Header 传递，Refresh Token（7 天）存储于 HttpOnly + Secure + SameSite=Lax Cookie。
- CSRF 防护：API 使用 Authorization Header 认证，天然免疫 CSRF；Refresh 接口额外校验 Origin/Referer。
- 密码使用 bcrypt 哈希存储，不记录明文密码。
- Refresh Token 使用 Rotation 策略，检测到已吊销 Token 被复用时立即吊销该用户所有 Token。
- 登录、注册、评论、发布、上传图片、Token 刷新都需要限流。

## 3. 匿名发布规则

匿名发布只影响前台展示：

- 前台显示“匿名用户”。
- 用户主页不公开展示匿名内容，除非用户选择展示。
- 后台仍可看到真实作者，用于审核和风控。
- 作者本人可在个人中心看到自己的匿名内容。

## 4. 隐私保护

发布前必须提示用户检查：

- 真实姓名。
- 手机号。
- 地址。
- 订单号。
- 聊天记录。
- 学校、公司、住址等定位信息。
- 未经同意的他人正脸照片。

MVP 功能：

- 发布前隐私确认勾选。
- 支持匿名。
- 支持隐藏具体价格。
- 支持隐藏关系时长。
- 图片 alt 和说明不自动公开敏感信息。

P2 功能：

- 人脸识别提醒。
- 地址、订单号 OCR 检测。
- 聊天记录截图自动风险提示。

## 5. 内容审核状态

| 状态 | 触发 |
| --- | --- |
| `draft` | 用户保存草稿 |
| `pending_review` | 用户提交发布 |
| `published` | 审核通过 |
| `rejected` | 审核驳回 |
| `hidden` | 作者主动隐藏 |
| `removed` | 平台下架 |

## 6. 审核规则

必须驳回或下架：

- 明显广告、导流、返利、商家软文。
- 盗图、搬运他人内容。
- 暴露隐私。
- 攻击、歧视、低俗内容。
- 违法、危险、诱导伤害行为。
- 恶意标题党或虚假内容。

可以要求修改：

- 图片模糊或无关。
- 标签明显错误。
- 内容过短，无法提供参考价值。
- 标题不清楚。
- 缺少对方反应或建议。

## 7. 举报体系

举报目标：

- 礼物故事。
- 评论。
- 用户。

举报原因：

- 隐私泄露。
- 广告导流。
- 盗图搬运。
- 人身攻击。
- 歧视低俗。
- 虚假内容。
- 其他。

处理动作：

- 忽略。
- 隐藏评论。
- 驳回故事。
- 下架故事。
- 禁言用户。
- 封禁用户。

## 8. 反广告策略

- 新用户限制外链发布。
- 评论区禁止重复发布联系方式。
- 标题和正文检测电商导流词。
- 商品链接默认不展示，P1 后可考虑转成“购买渠道说明”，但不作为核心入口。
- 后台提供“疑似广告”筛选。

## 9. 反炫富策略

- 前端弱化具体价格展示，优先展示预算区间。
- 首页固定展示低预算高心意。
- 点赞命名为“很用心”，不叫“壕”或“高级”。
- 排序不提供“最贵”。
- 精选内容优先看情绪价值、真实反应和可复用性。

## 10. 速率限制

| 行为 | 限制建议 |
| --- | --- |
| 登录 | 5 次 / 10 分钟 |
| 注册 | 3 次 / 小时 / IP |
| 发布故事 | 10 篇 / 天 / 用户 |
| 评论 | 60 条 / 小时 / 用户 |
| 举报 | 30 次 / 天 / 用户 |
| 图片上传 | 100 张 / 天 / 用户 |

## 11. 审计日志

必须记录：

- 管理员审核动作。
- 用户封禁和解封。
- 标签合并。
- 内容下架和恢复。
- 举报处理。
- 登录失败频繁事件。

审计日志不允许普通管理员删除。
