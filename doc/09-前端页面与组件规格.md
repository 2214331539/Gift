# 前端页面与组件规格

## 1. 设计原则

Gift 是真实礼物故事社区，不是电商导购页。因此前端设计应围绕“故事、关系、场景、反应和参考价值”组织，而不是围绕商品价格和购买按钮。

原则：

- 信息密度适中，便于扫描和比较。
- 视觉风格温暖但不过度装饰。
- 首页第一屏必须提供搜索入口。
- 卡片标题优先展示故事化表达。
- 预算用区间呈现，避免价格攀比。
- 移动端必须可用。
- 发布流程用问题引导，不给用户一个空白大编辑器。
- 管理后台保持高密度、工具化、可批量处理。

## 2. 响应式断点

建议：

| 断点 | 宽度 |
| --- | --- |
| mobile | < 640px |
| tablet | 640px - 1024px |
| desktop | > 1024px |

布局原则：

- 移动端筛选使用抽屉。
- 桌面端筛选使用侧栏。
- 礼物卡片移动端单列，桌面端 3-4 列。
- 详情页移动端图片在上，内容在下；桌面端可使用主内容 + 侧栏操作。

## 4. 首页规格

### 4.1 页面模块

```text
Header
HeroSearch
QuickFilters
FestivalCountdown
FeaturedStories
HotScenarios
LowBudgetSection
LatestStories
FailureCases
ShareCTA
Footer
```

### 4.2 主搜索区

文案：

```text
你想送给谁？TA 最近发生了什么？
```

占位示例：

```text
女朋友 生日 预算300 喜欢拍照 最近压力大
```

交互：

- 回车或点击搜索进入 `/stories?q=...`。
- 快捷标签点击进入 `/stories?recipient=...`。
- 移动端搜索框宽度占满。

### 4.3 首页内容卡片

展示优先级：

1. 图片。
2. 故事标题。
3. 送给谁 / 场景 / 预算。
4. 一句反应或摘要。
5. 收藏、很用心、被参考次数。

## 5. 礼物卡片组件

组件名建议：

```text
StoryCard
```

Props：

```ts
type StoryCardProps = {
  id: string;
  slug: string;
  title: string;
  summary?: string;
  coverImageUrl: string;
  recipientType: string;
  occasion: string;
  budgetRange: string;
  preparationTime?: string;
  giftType?: string;
  saveCount: number;
  reactionCount: number;
  forkCount: number;
  isSaved?: boolean;
};
```

状态：

- 默认。
- Hover。
- 图片加载中。
- 图片加载失败。
- 已收藏。

禁止：

- 把价格作为最大字号。
- 把购买按钮作为主按钮。
- 卡片只展示商品名。

## 8. 发布礼物页规格

### 8.1 流程

```text
Step 1 基础信息
Step 2 上传照片
Step 3 礼物故事
Step 4 标签和预算
Step 5 隐私设置
Step 6 预览提交
```

### 8.2 表单组件

组件：

- PublishStepper
- RecipientSelector
- OccasionSelector
- ImageUploader
- StoryQuestionInput
- TagPicker
- BudgetSelector
- PrivacySettings
- ContentCompleteness
- PublishPreview

### 8.3 问题引导

不要让用户面对空白编辑器。用问题组织输入：

- 你送给了谁？
- 是什么场景？
- 这份礼物具体包含什么？
- 你为什么想到送这个？
- 准备了多久？
- 花了多少钱？
- TA 收到后什么反应？
- 有没有什么建议给后来的人？

### 8.4 图片上传

要求：

- 至少 1 张。
- 最多 9 张。
- 支持拖拽和点击选择。
- 支持设置封面。
- 上传前展示隐私提醒。
- 上传失败可重试。

### 8.5 内容完整度

组件显示：

```text
当前完整度：72%
```

补充建议：

- 对方反应
- 踩坑提醒
- 准备时间

完整度不阻止提交，但缺少必填字段必须阻止提交。

## 9. 登录注册页面

MVP：

- 邮箱注册。
- 邮箱登录。
- 密码登录或验证码登录二选一。

交互：

- 写操作触发登录弹窗或跳转。
- 登录成功后返回原页面并继续原操作。

## 10. 个人主页规格

页面模块：

- 用户基础信息。
- 温暖型贡献指标。
- 发布过的礼物。
- 收藏夹。
- 收到的感谢。

指标文案：

```text
分享了 12 个礼物故事
帮助了 86 个送礼的人
收到 24 条感谢
贡献了 5 条避坑提醒
```

避免：

- 粉丝排行榜。
- 炫耀式等级。
- 财富导向指标。

## 11. 管理后台前端规格

后台设计原则：

- 高密度。
- 操作清晰。
- 支持筛选和批量处理。
- 所有危险操作二次确认。

内容审核列表字段：

- 标题。
- 作者。
- 状态。
- 是否匿名。
- 图片数。
- 标签完整度。
- 疑似风险。
- 提交时间。
- 操作。

审核详情：

- 用户填写内容。
- 图片预览。
- 标签列表。
- 隐私风险提示。
- 操作按钮：通过、驳回、隐藏。
- 审核日志。

## 13. 可访问性

- 所有图片有 alt 或空 alt。
- 表单字段有 label。
- 错误信息与字段关联。
- 按钮有明确文本或 aria-label。
- 键盘可以完成核心操作。
- 色彩对比度满足常规可读性。

## 14. 前端验收标准

- 首页、列表、详情、发布、个人主页在移动端和桌面端可用。
- 用户可以从首页完成搜索并进入详情。
- 礼物卡片信息完整，不像商品导购卡。
- 发布流程能保存并提交完整内容。
- 表单错误提示清晰。
- 登录后能回到原操作。
- 图片上传有加载、失败和重试状态。
- 管理后台能完成审核和举报处理。
- 所有核心页面有 loading、empty、error 状态。
